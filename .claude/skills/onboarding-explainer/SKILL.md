# 非技術者への説明ガイド

目的: 聞き手が読んでいるうちに理解が深まり、技術・会計スキルが自然に増えること。やさしくするが、正確さは削らない（＝わかりやすさは正確さへの"足し算"）。

## 説明の型
1. **たとえ → 正確な言葉** の順。まず日常の比喩で直感を作り、その後に正しい用語を置く。
2. **一度に一つ**の新概念。積み上げ式に。
3. **なぜ大事か**を必ず添える。「これは○○を防ぐため」と用途に結びつける。
4. 略語は初出で展開。専門語は初出でかっこ書きの一言解説。
5. 最後に**理解の確認**を一言（「ここまでで詰まる所はありますか」）。

## この領域の用語 ── たとえ付き
- **カットオフ（期間帰属）** — 「その売上は今期か、来期か」の線引き。カレンダーで日付をまたぐ荷物をどちらの月に数えるか、と同じ。売上不正の多くは「期日までに数字を作る」ために、この線を動かすことで起こる。
- **収益認識（いつ売上を計上してよいか）** — 商品を渡す約束（履行義務）を果たして初めて売上にできる、というルール。前払いを受けても、渡すまでは"預かり"に近い。
- **アサーション（監査要点）** — 「その売上、本当に(1)起きた？ (2)漏れなく計上した？ (3)金額正しい？ (4)正しい期間？」という監査の問いのセット。各チェックがどの問いに答えるかを常に紐づける。
- **循環取引** — A社→B社→C社→A社と、同じ財や資金をぐるぐる回して、実需がないのに売上を水増しする手口。バケツリレーで水を回しても総量は増えないのに、各人が「渡した」と記録する状態。
- **統制無効化（management override）** — ルールを作った人（経営者）自身がルールを破れてしまう問題。門番が共犯だと門は意味をなさない。だから独立した監査人・監査役が要る。
- **仕訳テスト** — 手作業で入れた会計記録、特に決算間際・経営者による調整を狙って点検すること。自動処理でなく"手で足した数字"に不正が紛れやすい。
- **異常検知 (anomaly detection)** — 健康診断で基準値を外れた項目に印がつくのと同じ。多数派から外れた取引を機械が見つける。
- **HITL（Human-in-the-Loop）** — AIは候補を出すだけ、確定は必ず人間。自動運転でも最後はドライバーが判断するのと同じ。
- **ファネル（ふるい分け）** — 砂金採りのように、まず機械で大量を粗くふるい、残った"怪しいものだけ"を人とAIで丁寧に調べる。全件を丁寧に見るのは非現実的なので、この二段構えにする。
- **プロンプトインジェクション** — 契約書やメールに紛れ込ませた「命令のふりをした文章」。それに釣られてAIが勝手に動かないようにする、という話。
- **WORM＋ハッシュチェーン** — 一度書いたら書き換えできない記録。1文字でも変えると鎖が切れて改ざんが分かる。
- **説明可能性 (explainability)** — 「なぜそう判定したか」を必ず言えること。監査では結論だけでなく理由が要る。

## 調整
聞き手の反応を見て深さを変える。会計に強く技術に不慣れな監査人なら会計語彙を土台に技術を橋渡しし、逆なら会計語彙をたとえで補う。専門性が見えたら比喩は最小限に。
