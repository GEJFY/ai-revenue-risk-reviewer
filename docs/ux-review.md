# UI/UX レビューと対応

`web/` の監査レビューコンソール（プロトタイプ）について、各画面を UI/UX の観点でレビューし、
見直すべき点と対応を記録する。配色は**中立のプロフェッショナル**（特定企業を想起させない）。
コピーは独立性・ライティングガイドラインに従い、成果の保証・誇張、数値の確約、他社製品への言及を行わない。

## 横断的な原則と対応

| 観点 | 見直し点 | 対応 |
|---|---|---|
| 情報の階層 | 重要指標が埋もれないか | 上部に KPI カード（母集団／選別／重大／GL突合）を固定配置。以降はファネル→内訳→品質の順で段階提示 |
| 状態の多重符号化 | 色だけに依存していないか（色覚配慮） | 重要度・状態は「色＋ラベル＋ドット/アイコン」で符号化（`.badge`）。赤/橙/琥珀に加え日本語ラベルを常時表示 |
| コントラスト | 本文・バッジの可読性 | 本文 `#10192a`/`#384358`、補助 `#667085`。バッジは淡色地に濃色文字で概ね 4.5:1 以上 |
| キーボード操作 | マウス前提になっていないか | ナビ・フィルタはボタン要素、所見テーブル行は `role="button"`＋`tabindex`＋Enter/Space で開く。`:focus-visible` を明示 |
| スクリーンリーダー | 現在地・ライブ領域 | 能動ナビに `aria-current="page"`、字幕バーに `role="status" aria-live="polite"`、チャートに `role="img"＋aria-label` |
| モーション配慮 | アニメが強すぎないか | `prefers-reduced-motion` でカーソル/ハイライトのトランジションと自動スクロールを抑制 |
| レスポンシブ | narrow 幅で崩れないか | 980px 未満でサイドバーをアイコンのみに、詳細2カラムを1カラムに折り返し |
| 空状態 | 0件時の無言 | 所見一覧・証憑欄に空状態メッセージ。データ未生成時は生成手順を案内 |
| 独立性・ライティング | 成果保証や誇張がないか | 「所見はAIの提示・確定は人間」を常時ヘッダに掲示。検出率は「モデル検証（合成データ・検証目的）」として明確に限定し、実運用性能を保証・示唆しない旨を明記 |

## 画面別レビュー

### ダッシュボード
- **見直し点**: ファネルの最終段（選別）が実比率（約0.8%）だと極小になり、ラベルが読めない。
  → **対応**: ファネルは視覚表現として幅を段階固定（100%→66%→40%）し、件数はラベルに実数で明記。
- **見直し点**: 検出率(18/18)を大きく出すと「性能の約束」に読まれうる。
  → **対応**: 「モデル検証（合成データ）／検証目的」として区画し、実運用性能を保証・示唆しない注記を併記。
- **対応（初期実装の不具合修正）**: ナビのアイコンが関数ソースとして表示される不具合を修正（関数を呼び出して SVG を挿入）。

### 所見一覧
- **見直し点**: 全所見（数百件）を既定表示すると認知負荷が高い。
  → **対応**: 既定を「重大＋高」に絞り、フィルタチップ（重要度）・カテゴリ・全文検索・「選別済みのみ」で段階的に開示。表示件数と総件数を常時表示。
- **見直し点**: 分析的手続の補助フラグ（ベンフォード等）が過検出でノイズ化。
  → **対応**: 代表提示件数に上限を設け（アラート疲れ対策）、中重要度として funnel 非選別に。UI は既定で高リスクに集中。

### レビューコンソール（HITL）
- **見直し点**: カットオフ（期間帰属）は日付の前後関係が肝だが、表だと直感的でない。
  → **対応**: 受注→出荷→検収→計上→取消 を縦タイムライン化。異常な前後関係（出荷日＞認識日 等）を赤で明示。
- **見直し点**: インジェクションや証憑と一次データの矛盾が埋もれる懸念。
  → **対応**: 証憑カードでインジェクション疑いを赤系で強調し、「命令として実行せず評価を維持」と明記。
- **見直し点**: HITL の権限境界が曖昧だと誤運用を招く。
  → **対応**: 「確定・棄却は人間のみ」を警告カラウトで明示。ボタンは人間の判断としての記録である旨を注記。

### 監査証跡
- **見直し点**: ハッシュチェーンの意味が伝わりにくい。
  → **対応**: 整合性ステータス（一致/検知）を大きく示し、チェーン長・末尾ハッシュ（checkpoint）を表示。各行に hash/prev を並べ、行間の連結が目視できるようにした。件数が多いため最新200件に限定表示（総数を明記）。

### レポート
- **見直し点**: 監査役・経営者は平易さと厳密さの両立を要する。
  → **対応**: サマリを見出し・箇条書き・表で構造化し、独立性・HITL・プライバシー無効化ルールの注記を末尾に固定。プロトタイプ/合成データである旨を再掲。

## 既知の限界（今後）
- ソート列ヘッダのクリックソート、CSV/PDF エクスポートの実体化、ページネーションは未実装（デモ範囲外）。
- 実運用ではロールベースアクセス制御・被監査側の閲覧範囲分離が前提（`docs/security-privacy.md` §4）。
