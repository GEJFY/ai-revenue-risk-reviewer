# UI/UX レビューと対応

`web/` の監査レビューコンソール（プロトタイプ）について、UI/UX を厳格にレビューし、見直し点と対応を記録する。
配色は**中立のプロフェッショナル**。コピーは独立性・ライティングガイドラインに従い、成果の保証・誇張、
数値の確約、他社製品への言及を行わない。所見はAIの提示・確定は人間（HITL）。

## 世界水準デザインレビュー（各画面）と全面リデザイン

各画面のスクリーンショットとコードを、世界水準のプロダクトUIデザイナー相当の観点で**極めて厳格に**レビューし、
**73件の指摘（critical 9 / major 39 / minor 25）**を洗い出した。指摘は「角丸・pill・影・グラデ・絵文字の
過剰」「8pxリズムの欠如」「タイポ階層の弱さ」「tabular numerals 未徹底」「データビズの粗さ（軸・基線・整列の欠如）」
「暖色の散在」に集約された。これらを踏まえ、デザインシステムを全面刷新した。

### 適用したデザインシステム（styles.css）
| 項目 | 変更前 | 変更後（対応） |
|---|---|---|
| 角丸 | カード10px・pill20px・混在(4/6/7/8/10/20) | カード2px・タグ1px・バー0。pill全廃、矩形タグへ |
| 面の分離 | ドロップシャドウ（全カード） | 影を全廃し**ヘアライン罫のみ** |
| 余白 | 11/13/18/22/26px 混在 | **8pxリズム**（4/8/16/24/32） |
| タイポ | 10.5〜15.5px・weight 650/720/730 | モジュラースケール（11/12/13/14/16/20/28）・weight 400/500/600/700 |
| 数値 | 一部のみ tabular | **全域 tabular numerals**（body 既定 tnum/lnum、和欧アキ palt） |
| 配色 | 赤/橙/黄土＋紫2種＋青＋緑が散在 | 中立グレースケール＋抑制アクセント1色。重要度は**左カラーバー＋ラベル**（面塗り廃止） |
| ロゴ/ナビ | グラデ地・塗りpillのアクティブ | 単色ロゴ・アクティブは**左2pxアクセント罫** |

### 画面別の主な対応
- **ダッシュボード** — KPIをグリッド配置（アイコンfloat廃止・塗り枠廃止）。横棒に**基線ヘアライン＋右端固定の値カラム**を追加し数値左端を整列、rx=0。ファネルはrx=0＋「面積は厳密比でない」注記。ドーナツを細身化。重要度・GL・エージェントの色符号を中立化。
- **所見一覧** — 重要度を左カラーバー＋ラベルに。スコアを**固定幅グリッド（数値＋矩形バー）**で右端整列。ルールをmonoテキスト化（紫pill廃止）。HITL状態はドット＋テキスト、選別は極小マーカー、hoverを無彩色化。行密度を向上。
- **レビューコンソール** — 絵文字⚠を廃しタイムラインのドット塗り分け＋「逸脱」ラベルへ。SHAPバーを中立色化＋注記。証憑を**入れ子カード→ヘアラインリスト**へ。ルール行を`rule-row`で整列（最終行の二重罫を除去）。ヘッダのスコアに統合注記、HITLボタンを整理（棄却は中立枠）。
- **監査証跡** — 整合性カードを角丸・tint廃止し**左状態罫＋右3メトリクス（tabular右揃え）**へ。テーブルを`table-layout:fixed`＋colgroupで列固定。**prev_hash と hash を隣接配置**し行間の連鎖を目視可能に（prevは淡色）。同一時刻は「〃」で圧縮。
- **レポート** — 引用の青塗り箱を**ヘアライン罫の引用**へ。**要約の機械的60字切りを1文境界に修正**（`/[RULE]`断片の露出を解消）。金額に¥、`track_a/third_line`・ルールIDを**人間可読名**に解決、アサーションを日本語化、数値を tabular・見出しの折返しを防止。

## 横断的な UX 原則（維持・強化）
- 状態は「色＋ラベル＋形」で多重符号化（色覚配慮）。重要度は左バー＋日本語ラベル。
- キーボード操作（テーブル行 `role=button`＋Enter/Space、`aria-current`、`:focus-visible`）、字幕 `aria-live`、チャート `role=img`。
- `prefers-reduced-motion` でアニメ・自動スクロールを抑制。980px 未満でサイドバー折りたたみ・詳細1カラム化。
- 独立性・ライティング: 「所見はAIの提示・確定は人間」を常時掲示。検出率は「モデル検証（合成データ・検証目的）」として限定し、実運用性能を保証・示唆しない。プロトタイプ/合成データを明記。
