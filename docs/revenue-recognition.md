# 収益認識・監査アサーション仕様

売上リスク分析を「実務レベルで意味のある」ものにする土台。検知ルールを、会計監査の言語である **収益認識の5ステップモデル**（収益認識に関する会計基準）と **財務諸表アサーション** に対応づける。これにより、所見が「監査人が何を検証すべきか」に直結する。

## 1. なぜ売上が最重要リスクなのか

監査基準委員会報告書240（財務諸表監査における不正）は、**収益認識には不正リスクが存在すると推定する**（反証可能な推定）。売上・収益は経営者が財務諸表を歪める際の主要な手段であり、これは統制を無効化できる立場の者による不正（management override）である点が、末端従業員による経費不正と根本的に異なる。したがって本ツールは、統制の逸脱そのものに加え、統制をすり抜ける取引の実在性・期間帰属・仕訳（トップサイド）を重点的に評価する。

## 2. 収益認識の5ステップと検知の対応

| ステップ | 内容 | 主なリスクと対応カテゴリ |
|---|---|---|
| 1 契約の識別 | 顧客との契約を識別 | 契約実体の欠如・架空契約 → FICT, CUST |
| 2 履行義務の識別 | 別個の履行義務を識別 | 複数履行義務の未分解 → RECOG-005 |
| 3 取引価格の算定 | 変動対価等を含め価格を算定 | 変動対価（返品・リベート）の見積り → RECOG-006, RETURN |
| 4 取引価格の配分 | 各履行義務へ配分 | 配分の欠如・恣意 → RECOG-005 |
| 5 履行義務の充足時に認識 | 支配移転・充足に応じ認識 | 前倒し計上・進捗操作・総額純額 → RECOG-001〜004, CUTOFF |

## 3. 財務諸表アサーション（会計監査の言語）

各ルールは、リスクに晒すアサーションを `assertion` として持つ。売上取引・売掛金残高の主要アサーション:

| アサーション | 売上における意味 | 主な対応カテゴリ |
|---|---|---|
| occurrence（発生・実在性） | 計上した売上が実際に発生したか | FICT, CIRC, JE |
| completeness（網羅性） | すべての売上が計上されているか（簿外・計上漏れ） | CUTOFF-005, RETURN-002, RECOG-006 |
| accuracy（正確性） | 金額が正しいか | PRICE, RECOG-002, JE |
| cutoff（期間帰属） | 正しい会計期間に計上されたか | CUTOFF, JE-002/006, RETURN |
| classification（分類） | 適切な勘定・区分か | RECOG-004, CIRC-004, JE-004 |
| valuation（評価） | 売掛金が回収可能な額で評価されているか | CREDIT, RETURN, RECOG-006 |
| rights_obligations（権利と義務） | 売掛債権の権利が実在するか | CREDIT-001 |
| presentation（表示・開示） | 適切に表示・開示されているか（関連当事者等） | CIRC-005, CUST, COMPL |

**カットオフ（期間帰属）が売上リスクの主軸**である理由: 不正・利益調整の多くは「ある数字を、ある期日までに作る」ために起こる。したがって、受注・出荷・検収・請求・収益認識・入金・返品の各日付を分離して三角照合することが、売上分析の生命線になる（`SalesTransaction` の日付フィールド設計はこのため）。

## 4. 仕訳テスト（JE Testing）と統制無効化

監基報240 は、統制無効化リスクへの対応として**仕訳入力（journal entries）と修正のテスト**を求める。売上の文脈では、手動・トップサイドの売上仕訳、決算日近接の調整、上位者による起票、異例な勘定組合せ、バックデートに着目する（JE カテゴリ）。これは「統制が有効でも、統制を無効化できる者による不正は残る」という前提に立つ、売上分析の中核をなす手続。

## 5. 内部監査・継続モニタリングでの使い方

会計監査だけでなく、内部監査（3線）や2線の継続モニタリングでも同じルール・アサーション枠組みを使える。違いは頻度と目的（`docs/governance.md` の配備レイヤ）。継続モニタリングでは、期末を待たず、カットオフ・JE・与信・返品の指標を準リアルタイムで監視し、期末の駆け込みや調整を早期に把握する。
